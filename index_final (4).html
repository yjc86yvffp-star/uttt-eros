Evaluar los Diseños Frente a los Requisitos

Junio 24, 2026  |  UTTT EROS  |  Ingeniería de Software

Objetivo: Validar si el software es fácil de usar y si realmente resuelve los problemas del usuario. Es la fase crítica para el control de calidad, evaluando el sistema al final (o durante) su desarrollo para confirmar que cumple con las necesidades y el uso previsto por el usuario final.

1. Pruebas de Usabilidad con Usuarios Reales

Consiste en observar a personas del público objetivo interactuando con el software para descubrir problemas de diseño en tiempo real.

Ejemplo práctico: Se reclutan 5 adultos mayores (65–75 años) para una app de telemedicina. Se les pide: "Agenda una cita con el cardiólogo para mañana a las 4:00 PM." El resultado revela que el icono de corazón en "Cardiología" se confunde con "Favoritos".

Guion de Tareas para Pruebas de Usuario

  • Tarea 1: Registra una cuenta nueva con tu correo electrónico.
  • Tarea 2: Encuentra la sección de configuración y cambia tu contraseña.
  • Tarea 3: Crea un nuevo proyecto y asígnale un presupuesto.
  • Tarea 4: Invita a un colaborador usando su dirección de correo.
  • Tarea 5: Exporta el reporte del proyecto en formato PDF.

Cierre: ¿Cuál fue la tarea más difícil y por qué? ¿Qué cambiarías de la interfaz de forma inmediata?

2. Evaluación Heurística por Expertos

Un especialista UX analiza la interfaz usando las 10 heurísticas de Jakob Nielsen, sin necesidad de interactuar con usuarios finales. Detecta violaciones de buenas prácticas de diseño antes del lanzamiento.

Lista de Verificación Heurística

  • El sistema muestra indicador de carga durante procesos largos.
  • El usuario puede deshacer (Ctrl+Z) la última acción realizada.
  • Los botones con la misma función usan el mismo color.
  • El sistema valida los campos del formulario antes de enviar.
  • Los mensajes de error explican cómo solucionar el problema.
  • Existen atajos de teclado para las funciones más repetitivas.
  • La interfaz se adapta correctamente a dispositivos móviles.

3. Análisis de Tasas de Error y Tiempos de Tarea

Recolección de datos cuantitativos para medir eficiencia y eficacia de forma objetiva. Ejemplo: en una app de telemedicina, el 45% de usuarios busca recetas en "Configuración" en lugar de "Mis Documentos", con un tiempo promedio de 4 minutos (meta: <1 min).

Métricas de Éxito del Software

Adopción y Uso

  • Tasa de conversión de registro: % de visitantes que crean cuenta.
  • Usuarios activos diarios (DAU): Usuarios únicos por día.
  • Tiempo de retención: Días de uso continuo del software.
Tasa de Conversión = (Registros Completados / Visitantes Totales) × 100

Ejemplo: 150 registros / 1,500 visitantes × 100 = 10%

Experiencia UX

  • Tasa de éxito de tareas: % de tareas completadas sin errores.
  • Tiempo por tarea: Segundos en completar un flujo específico.
  • CES (Puntuación de Esfuerzo del Cliente): Facilidad percibida por el usuario (escala 1–7).
Tasa de Éxito = (Tareas completadas con éxito / Total de intentos) × 100 CES promedio = Suma de puntuaciones / Número de respuestas

Técnicas y Rendimiento

  • Tiempo de carga inicial: Segundos en mostrar la pantalla principal.
  • Tasa de error de API: % de peticiones fallidas al servidor.
  • Tiempo de respuesta del servidor: Milisegundos en procesar solicitudes.
Tasa error API = (Solicitudes fallidas / Solicitudes totales) × 100 T_total = T_DNS + T_TCP + T_SSL + T_Procesamiento + T_Red

Resumen del Impacto en el Diseño

  • Evaluación heurística: Limpia los errores de diseño más obvios antes de lanzar el producto.
  • Pruebas con usuarios: Aportan empatía y contexto real de uso.
  • Tasas de error y tiempos: Proveen la prueba matemática de que el software funciona bien.
